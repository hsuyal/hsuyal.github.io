今天用了一个新的东西做了一个toolbar，我之前的toolbar都是自己写的view，tab页面也都是自己写的view，后来tabs页面换成了viewpager，但是toolbar一直还是用的view，今天我给改成了android.support.v7.widget.Toolbar做toolbar

Toolbar是一个专门做tool的东东，其中封装了很多好东西，比我之前自己写确实要方便很多。再加上PagerSlidingTabStrip，Fragtment和Viewpager基本上一个app框架就搭好了，特别是在包含好几个tab页面的app中，我现在记录下我写的过程，以免后来忘记供自己随时查阅。

android.support.v7中几个用到的类

  1. ActionBarDrawerToggle官方的解释是：This class provides a handy way to tie together the functionality of
    {@link android.support.v4.widget.DrawerLayout} and the framework ActionBar to implement the recommended design for navigation drawers.
  2. ShareActionProvider分享控件
  3. SearchView搜索控件，功能很强大

声明style文件

Toolbar尽量写在一个单独的文件中，这样其他的xml文件可以随时包含该toolbar；这是我的toolbar.xml文件

<style name="AppTheme" parent="Theme.AppCompat.Light.DarkActionBar">
    <!-- Customize your theme here. -->
    <item name="colorPrimary">@color/colorPrimary</item>
    <item name="colorPrimaryDark">@color/colorPrimaryDark</item>
    <item name="colorAccent">@color/colorAccent</item>
</style>
<style name="AppTheme.NoActionBar">
    <item name="windowActionBar">false</item>
    <item name="windowNoTitle">true</item>
</style>
<style name="AppTheme.AppBarOverlay" parent="ThemeOverlay.AppCompat.Dark.ActionBar" />
<style name="AppTheme.PopupOverlay" parent="ThemeOverlay.AppCompat.Light" />

这是我们要用上的文件，至于toolbar各个不同部位的颜色，可以自己随意处理。注意如果在android5以上的话需要在vlues-21下面的styles.xml文件里面复制一份一样的。我是这么做得，方便，也可以通过设置parent的方法吧，我还不会，菜鸟一个，只会复制撒。
Toolbar尽量写在一个单独的文件中，这样其他的xml文件可以随时包含该toolbar；这是我的toolbar.xml文件

<?xml version="1.0" encoding="utf-8"?>
<android.support.v7.widget.Toolbar xmlns:android="http://schemas.android.com/apk/res/android"
xmlns:app="http://schemas.android.com/apk/res-auto"
android:id="@+id/toolbar"
android:layout_width="match_parent"
android:layout_height="?attr/actionBarSize"
android:background="?attr/colorPrimary"
android:minHeight="?attr/actionBarSize"
app:popupTheme="@style/ThemeOverlay.AppCompat.Light"
app:theme="@style/ThemeOverlay.AppCompat.ActionBar"/>

在activity_main.xml文件中包含就可以了，可以随自己喜欢摆放toolbar的位置

<include layout="@layout/toolbar"/>

这样基本上该toolbar就会显示了，接下来要做的是侧滑菜单

侧滑菜单使用android.support.v4.widget.DrawerLayout

android.support.v4.widget.DrawerLayout这个做侧滑的类xml文件如下：

    android:id="@+id/drawer"
    android:layout_width="match_parent"
    android:layout_height="match_parent" >

    <!-- 内容界面 -->
    <!--<include layout="@layout/toolbar"/>-->
    <LinearLayout
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:orientation="vertical" >

        <com.astuetz.PagerSlidingTabStrip
            android:id="@+id/tabs"
            android:layout_width="match_parent"
            android:layout_height="70dip" />

        <android.support.v4.view.ViewPager
            android:id="@+id/pager"
            android:layout_width="match_parent"
            android:layout_height="match_parent" >
        </android.support.v4.view.ViewPager>
    </LinearLayout>

    <!-- 侧滑菜单内容 -->

    <LinearLayout
        android:id="@+id/drawer_view"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:layout_gravity="left"
        android:orientation="vertical"
        >

        <include layout="@layout/leftlayout"/>


    </LinearLayout>
</android.support.v4.widget.DrawerLayout>

其中关于PagerSlidingTabStrip我使用的是一个github上的一个开源库，不知道需不需要附链接，但我的博客阅读量非常少，应该没问题吧，如果有问题也可以联系我，刚开始写博客，很多都不懂的。

接下来是toolbar，可以给toolbar做一些设置，如toolbar.setTitle()，toolbar.setTitleTextColor()等也可以设置toolbar.setLogo()等等，亦可以在setSupportActionBar(toolbar);之后通过getSupportActionBar.setLogo()这样的方法去设置，注意该方法只能用在setSupportActionBar以后。

Toolbar toolbar = (Toolbar) findViewById(R.id.toolbar);
setSupportActionBar(toolbar);

getSupportActionBar().setDisplayHomeAsUpEnabled(true);//设置点击home的样式
mDrawerLayout = (DrawerLayout) findViewById(R.id.drawer);
//以下几行代码就是侧滑的关键代码
    mDrawerToggle = new ActionBarDrawerToggle(this, mDrawerLayout, toolbar, R.string.drawer_open,
            R.string.drawer_close);
    mDrawerToggle.syncState();
    mDrawerLayout.setDrawerListener(mDrawerToggle);

ShareActionProvider的使用

我的menu里边含有三个item如下：

<menu xmlns:android="http://schemas.android.com/apk/res/android"
xmlns:app="http://schemas.android.com/apk/res-auto"
xmlns:tools="http://schemas.android.com/tools" tools:context=".MainActivity">
<item
    android:id="@+id/ab_search"
    android:orderInCategory="80"
    android:title="action_search"
    app:actionViewClass="android.support.v7.widget.SearchView"
    app:showAsAction="ifRoom"/>
<item
    android:id="@+id/action_share"
    android:orderInCategory="90"
    android:title="action_share"
    app:actionProviderClass="android.support.v7.widget.ShareActionProvider"
    app:showAsAction="ifRoom"
    >
</item>
<item android:id="@+id/action_settings"
    android:title="@string/action_settings"
    android:orderInCategory="100" app:showAsAction="never" />

</menu>

关于包含app:actionProviderClass=”android.support.v7.widget.ShareActionProvider和app:actionViewClass=”android.support.v7.widget.SearchView这种系统自带的东东的在onOptionsItemSelected方法中是无法获取id的，但是如果自己写就可以获取到id，如给他设置一个一样的android:icon就可以自己定义分享功能。

我们需要在onCreateOptionsMenu这个方法中做处理：代码如下

@Override
public boolean onCreateOptionsMenu(Menu menu) {
    // Inflate the menu; this adds items to the action bar if it is present.
    //对ShareActionProvider的处理,MenuItemCompat是MenuItem的兼容版
    getMenuInflater().inflate(R.menu.menu_main, menu);
    mShareActionProvider = (ShareActionProvider) MenuItemCompat.getActionProvider(menu.findItem(R.id.action_share));
    Intent intent = new Intent(Intent.ACTION_SEND);
    intent.setType("text/*");
    mShareActionProvider.setShareIntent(getShareIntent());

    //以下是对SearchView的处理
    MenuItem search_item = menu.findItem(R.id.ab_search);
    SearchView mSearchView = (SearchView) MenuItemCompat.getActionView(search_item);

    return super.onCreateOptionsMenu(menu);
}

private Intent getShareIntent(){
    Intent intent = new Intent(Intent.ACTION_SEND);
    intent.putExtra(Intent.EXTRA_TEXT, "Share Words");
    intent.setType("text/plain");
    return intent;
}

需要注意的是：mShareActionProvider.setShareIntent(getShareIntent());可能找不到setShareIntent这个方法，这个问题困扰我蛮久，但是很简单，因为ShareActionProvider不止在v7中有，v4中也有，貌似非这两个类中也有一个，所以import的时候要特别注意，不要import错了。否则怎么都找不到撒。好了今天该回去了，貌似就是贴代码了，木办法，还是菜鸟一个，慢慢学吧。
